vue-basics

V-BIND / V-ON / V-MODEL

  • v-bind:{html document element name} = "{vue instance variable name}"

    • This is used for binding reactively and dynamically

    • In this case, the html document element name could be the child component’s instance variable name

    • {html document element name} in parent component is {vue instance variable name} in child component

    • Without v-bind::

      • the value of the element as well as the child component’s instance variable does not vary reactively

      • the content within the quote signs will not be evaluated as javascript expression

  • v-on:{html document event name} = "{vue instance function name}"

  • v-model = "{name of vue instance variable that vary with vue instance function}"

SLOT

  • V-SLOT: v-directives within template component in parent component

  • SLOT: vue component name within child component

SLOT WITH V-BIND

  • PARENT: <{child component name}><template v-slot:{slot name} = "slotProps"> ... </template></{child component name}>

  • CHILD: <slot name = "{slot name}" v-bind:{element or property or event name} = "{an object key in slotProps object, or 'on' / 'attrs'}">

  • CAUTIOUS: slopProps is different from props, so slopProps should not be defined in script section

Is VI Editor Convenient for You?

  • The key idea is that users should discern two modes: command mode and insert mode

SUMMARIZATION

  • To enter overriding mode, press ‘i’ or ‘a’ with or without ctrl

  • To move around, press ‘h’, ‘j’, ‘k’, ‘l’

  • To move around by word, press ‘w’ or ‘b’

  • To erase a letter, press ‘x’

  • To erase a line, press ‘d’

  • To select a letter or a line, press ‘v’ or ‘V’

  • To copy, press ‘y’

  • To paste, press ‘p’

  • To fix a letter, press ‘r’

  • To insert letters, press ‘r’ with ctrl

  • gg / G

  • { / }

  • 0 / $

  • H / M / L

  • j / k

  • h / l

  • i / a

  • v / V

  • x / y / p

  • dd / D

그럼 진짜 흐름이 어떤식인가요?

  • vi abc.txt 명령으로 창열고.

  • 방향키랑 i , a , 이용해서 편집모드로 들어가서 편집 마구마구 하고.

  • 이러다가 잘라낼때는 그 위치에서 esc -> dd 눌러서 한줄 잘라내고.

  • 붙여 넣을 위치의 윗줄에서 p 눌러서 붙여 넣고

  • 또다시 i, a 이용해서 편집모드 들어갔다가 오타가 나오거나 하면 esc -> x 눌러서 글자 없애고 다시 i, a 이용해서 편집모드로 편집

  • 이러다가 다 작성하면 esc -> 콜론 (:) 누르고 -> wq 입력후 엔터 치면 저장후 종료까지 깔-끔.

CONVOLUTION 1D MULTILAYERS STRUCTURE IS TOTALLY DIFFERNT FROM 2D!

CONVOLUTION 1D MULTILAYERS STRUCTURE IS TOTALLY DIFFERNT FROM 2D In Case Of 2D Dataset

  • After a layer, all the channels are summed up as to each neuron in case of 2D

However, in case of 1D, for the next layer, all the channels are stacked in order to make a matrix

  • Of course, in this case, the input data is a set of matrices

  • In case of rank 3 dataset, conv1D as well as conv2D and conv3D are all possible.

  • All channels can be summed up or stacked down, which depends on which shape you want for the next traning layer.

json as a string

JSON as a special string

- JSON is not a special object in any language
- JSON is just a special format of string which could be stored in any txt file

JSON in javascript

- JSON could be parsed into just an ordinary javascript object or python-dictionary-like thing by `JSON.parse`
- 'JSON.stringify` could be used for converting a javascript object into a json string

JSON in python

- `json` package matters
- this package could be used for converting JSON string into neither a special object nor instance from a special python class, but just an ordinary python dictionary.

Github and SSH key setting

SSH?

  • Public / Private Key Pair using encryption technology like SHA256 widely accepted since 1970’s

  • For example, in case of Github, you can register your one public key at ACCOUNT SETTING, which means this works in relation of your github account (not project-wise) to your specific computer and you should register at every computer respectively

Key Generation

  • at `/home/user/.ssh/ using linux utility

  • `ssh-keygen -t rsa -b 4096 -C “hcjeong@kakao.com“

  • Enter passphrase

  • This process will produce two files which contains public key and private key respectively

  • Moreover, this will produce key fingerprint and key randomart image

  • You need to check Agent id by using eval "$(ssh-agent -s)"

  • ssh-add ~/.ssh/id_rsa: This will complete the process

  • You can copy public key by clip < ~/.ssh/id_rsa.pub
